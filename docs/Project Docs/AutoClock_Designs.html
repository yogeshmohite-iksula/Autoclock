HOW THE SYSTEM WORKS

Data Flow — One Backend, Five Roles

An employee logs work (with project + Jira task picked). It flows to one backend, gets tidied by a free deterministic parser, fans out to Jira / Sheet / Gmail, and feeds five role-based views.

Input surfaces Backend + AI Database Auto-synced outputs Role dashboards Admin config
1

🧩 Chrome Extension

Hourly nudge → full entry: pick project + Jira task, describe, save. EOD close — all in the popup.

1

💻 Web App

Same entry on a bigger screen, with the day timeline. Optional — extension can do it all.

▼ HTTPS / REST — both surfaces hit the same API ▼
2

⚙️ AutoClock Backend  +  Free Parser

A free deterministic parser tidies the description and totals durations (the employee already picked the ticket) → employee confirms a preview → backend writes everywhere at once. No paid AI.

▼ on "Confirm" ▼
3

🎯 Jira Worklogs

Posted to the exact ticket the employee picked — Clockwork displays them.

3

📊 Google Sheet

Row appended to the existing timesheet.

3

📧 Gmail Draft

EOD email drafted, ready to send.

4

🗄️ AutoClock DB

Every entry stored — powers all dashboards.

▼ stored data → role-based views ▼
5

👔 PM / Lead

Their team & project only — delivery metrics.

5

🏢 Management

All 60–70 people — utilization & cost.

5

📋 Operations

Weekly 40h compliance chase.

⚙

🛠️ Admin Console — sits across the whole system

Creates users & assigns roles, configures projects ↔ Jira mapping, manages integrations & global settings. Everything above runs on what Admin sets up.

Five roles, one source of truth: Employee logs · PM/Lead, Management & Operations read tailored dashboards · Admin configures it all. The extension and web app stay in sync because both read/write the same backend database.
